# 小泉动漫后端 - 部署说明

## 1. 环境要求

- Linux 服务器（CentOS 7+ / Ubuntu 18.04+ 等）
- MySQL 5.7+ 或 MariaDB 10.2+
- 无需安装 Go（使用编译好的可执行文件）

## 2. 编译步骤

### Windows 下编译 Linux 版本

在 Windows 开发环境下，在项目根目录执行：

```bash
./build_linux.sh
```

或者直接执行 Go 命令：

```bash
SET GOOS=linux
SET GOARCH=amd64
go build -o xiaoquan-backend
```

或者在 PowerShell 中：

```powershell
$env:GOOS = "linux"
$env:GOARCH = "amd64"
go build -o xiaoquan-backend
```

编译成功后会得到 `xiaoquan-backend` 文件（Linux 可执行文件）。

### Linux 本地编译

如果在 Linux 上直接编译：

```bash
chmod +x build_linux.sh
./build_linux.sh
```

## 3. 上传部署文件

将以下文件上传到 Linux 服务器：

- `xiaoquan-backend`（编译好的可执行文件）
- `config/config.yaml`（配置文件）

## 4. 配置文件

修改 `config/config.yaml` 中的数据库连接信息：

```yaml
database:
  host: "你的MySQL服务器地址"
  port: 3306
  user: "数据库用户名"
  password: "数据库密码"
  dbname: "数据库名"
  charset: "utf8mb4"
```

## 5. 创建 MySQL 数据库

在 MySQL 中创建数据库：

```sql
CREATE DATABASE xiaoquan CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 6. 上传到服务器

推荐目录结构：

```
/opt/xiaoquan-backend/
├── xiaoquan-backend
├── config/
│   └── config.yaml
└── uploads/ (自动创建)
```

## 7. 设置可执行权限

```bash
chmod +x xiaoquan-backend
```

## 8. 首次启动 - 创建管理员

首次启动时，先创建管理员：

```bash
# 启动服务
./xiaoquan-backend
```

另一个终端窗口（或使用 curl）调用初始化管理员接口：

```bash
curl -X POST http://localhost:8080/api/auth/init-admin
```

查看后端日志，记录下自动生成的管理员密码（用户名固定为 admin）。

## 9. 使用 systemd 管理服务（推荐）

创建 systemd 服务文件 `/etc/systemd/system/xiaoquan-backend.service`：

```ini
[Unit]
Description=XiaoQuan Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/xiaoquan-backend
ExecStart=/opt/xiaoquan-backend/xiaoquan-backend
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载配置
systemctl daemon-reload

# 设置开机自启
systemctl enable xiaoquan-backend

# 启动服务
systemctl start xiaoquan-backend

# 查看状态
systemctl status xiaoquan-backend

# 查看日志
journalctl -u xiaoquan-backend -f
```

## 10. 防火墙配置

开放 8080 端口：

```bash
# CentOS/RHEL
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload

# Ubuntu/Debian
ufw allow 8080/tcp
```

## 11. 验证部署

访问：`http://服务器IP:8080/api/content/list`

如果返回 JSON 响应说明部署成功。

## 12. 目录说明

- `uploads/`: 自动创建，存放上传的文件
- `config/config.yaml`: 配置文件

## 常见问题

### Q: 权限问题

确保运行服务的用户有读写配置文件和 uploads 目录的权限。

### Q: 端口被占用

修改 `config.yaml` 中的 server.port 配置。
