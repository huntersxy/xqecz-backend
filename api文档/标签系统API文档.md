
# 标签系统 API 文档

## 概述

标签系统已简化为一级标签，标签不单独存储在数据库中，而是直接附加在内容上。

**核心特点：**
- 标签不是必须项，可选填写
- 只有一级标签，无层级关系
- 标签直接存储在内容的 `tags` 字段中（JSON数组格式）
- 支持多标签，自动去重

---

## 标签字段说明

### Content 模型中的标签字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Tags | []string | 标签数组，存储标签名称列表 |

---

## 接口中的标签参数

### 1. 上传内容

**接口地址**：`POST /api/content/upload`

**标签参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tags | []string | 否 | 多个标签名称，支持多选 |

**请求示例：**
```bash
curl -X POST http://localhost:8080/api/content/upload \
  -F "title=测试内容" \
  -F "type=text" \
  -F "content=内容描述" \
  -F "tags=动画" \
  -F "tags=番剧" \
  -F "tags=推荐" \
  -b "session=xxx"
```

**响应示例：**
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "id": 1,
    "title": "测试内容",
    "type": "text",
    "tags": ["动画", "番剧", "推荐"],
    "created_at": "2026-05-07T14:00:00Z"
  }
}
```

---

### 2. 更新内容

**接口地址**：`PUT /api/content/:id`

**标签参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tags | []string | 否 | 更新后的标签列表，支持多选 |

**请求示例：**
```bash
curl -X PUT http://localhost:8080/api/content/1 \
  -F "tags=动画" \
  -F "tags=新番" \
  -b "session=xxx"
```

---

### 3. 获取内容列表

**接口地址**：`GET /api/content/list`

**标签筛选参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tag | string | 否 | 按标签名称筛选内容 |

**请求示例：**
```bash
curl "http://localhost:8080/api/content/list?tag=动画"
```

**响应示例：**
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "测试内容",
        "tags": ["动画", "番剧", "推荐"]
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  }
}
```

---

### 4. 随机推荐

**接口地址**：`GET /api/content/recommend`

**标签筛选参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tag | string | 否 | 按标签筛选推荐内容 |
| count | int | 是 | 返回条目数（最大50） |

**请求示例：**
```bash
curl "http://localhost:8080/api/content/recommend?tag=动画&count=10"
```

## 标签处理规则

### 1. 去重机制

系统会自动对标签进行去重，相同的标签只保留一个：

```json
// 输入
tags = ["动画", "番剧", "动画", "推荐"]

// 输出（自动去重后）
tags = ["动画", "番剧", "推荐"]
```

### 2. 空值过滤

空字符串标签会被自动过滤：

```json
// 输入
tags = ["动画", "", "番剧"]

// 输出（过滤空值后）
tags = ["动画", "番剧"]
```

### 3. 存储格式

标签以 JSON 数组格式存储在数据库中：

```json
["动画", "番剧", "推荐"]
```

## 内容详情中的标签

获取单个内容详情时，标签会作为数组返回：

**请求**：`GET /api/content/1`

**响应示例：**
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 1,
    "title": "测试内容",
    "type": "text",
    "content": "内容描述",
    "tags": ["动画", "番剧", "推荐"],
    "user_id": 1,
    "audit_status": "approved",
    "created_at": "2026-05-07T14:00:00Z",
    "updated_at": "2026-05-07T14:00:00Z"
  }
}
```

---

### 5. 获取全部标签

**接口地址**：`GET /api/content/tags`

**描述**：获取系统中所有内容使用的标签（自动去重）

**认证**：无需登录

**请求参数**：无

**响应示例：**
```json
{
  "code": 200,
  "message": "获取成功",
  "data": ["动画", "番剧", "推荐", "游戏", "新番"]
}
```

**响应字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| data | string[] | 所有唯一标签的数组 |

---

## 标签使用注意事项

1. **标签名称**：建议使用简洁的中文或英文标签，避免特殊字符
2. **标签数量**：建议每个内容不超过5个标签
3. **标签大小写**：系统不区分大小写（建议统一使用小写或中文）
4. **标签长度**：建议每个标签不超过20个字符
