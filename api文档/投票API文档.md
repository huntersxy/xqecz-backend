# 投票系统 API 文档

## 概述

本文档描述投票系统相关的 API 接口，包括投票的创建、查询、投票操作等。支持登录用户和路人（未登录用户）投票，路人投票使用 cookies 进行限制。

---

## 接口列表

### 1. 创建投票

**接口地址**：`POST /api/poll/create`

**描述**：创建新的投票

**认证**：需要登录 + 未被封禁

**请求方式**：`application/json`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 投票标题（1-200字符） |
| description | string | 否 | 投票描述 |
| options | string[] | 是 | 投票选项（至少2个选项） |

**响应示例**：

```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 1,
    "title": "测试投票",
    "description": "这是一个测试投票",
    "options": ["选项1", "选项2"],
    "vote_count": 0,
    "user_id": 1,
    "created_at": "2026-05-12T20:00:00Z"
  }
}
```

---

### 2. 获取投票列表

**接口地址**：`GET /api/poll/list`

**描述**：获取投票列表

**认证**：无需登录

**请求参数**：无

**响应示例**：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "测试投票",
        "description": "这是一个测试投票",
        "options": ["选项1", "选项2"],
        "vote_count": 5,
        "user_id": 1,
        "created_at": "2026-05-12T20:00:00Z"
      }
    ]
  }
}
```

---

### 3. 获取投票详情

**接口地址**：`GET /api/poll/:id`

**描述**：获取单个投票的详细信息，包括投票选项和结果

**认证**：无需登录

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | uint | 是 | 投票ID（URL路径参数） |

**响应示例**：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "poll": {
      "id": 1,
      "title": "测试投票",
      "description": "这是一个测试投票",
      "options": ["选项1", "选项2"],
      "vote_count": 5,
      "user_id": 1,
      "created_at": "2026-05-12T20:00:00Z"
    },
    "vote_counts": {
      "0": 3,
      "1": 2
    },
    "total_votes": 5,
    "my_vote": 0
  }
}
```

**字段说明**：
- `vote_counts`：各选项的投票数（键为选项索引）
- `my_vote`：当前用户的投票（未投票为 null，登录用户和路人都会返回）

---

### 4. 投票

**接口地址**：`POST /api/poll/:id/vote`

**描述**：对投票进行投票

**认证**：无需登录（支持路人投票）

**请求方式**：`application/json`

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| option_index | int | 是 | 选项索引（从0开始） |

**响应示例**：

```json
{
  "code": 200,
  "message": "投票成功",
  "data": null
}
```

**请求示例**：

```json
{
  "option_index": 0
}
```

**Cookies说明**：
- 对于未登录用户，会生成 `visitor_id` cookie（30天有效期）
- 每个 `visitor_id` 对同一投票只能投一次

---

### 5. 删除投票

**接口地址**：`DELETE /api/poll/:id`

**描述**：删除指定投票

**认证**：需要登录，且只能删除自己创建的投票（管理员可删除任意投票）

**请求参数**：无

**响应示例**：

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

## 路人投票机制

### 1. 原理

未登录用户（路人）投票通过以下机制实现：

1. **访客识别**：当路人首次访问时，服务器会生成一个唯一的 `visitor_id`（32位随机字符串）
2. **Cookie存储**：`visitor_id` 会存储在浏览器的 cookie 中，有效期30天
3. **投票限制**：同一 `visitor_id` 对同一投票只能投一次
4. **状态查询**：路人的投票状态会在 `GET /api/poll/:id` 接口的 `my_vote` 字段中返回

### 2. 前端处理

对于路人投票，前端不需要做特殊处理，只需要确保：

1. 发送正常的投票请求
2. 允许浏览器接收和发送 cookies
3. 正确处理 `visitor_id` cookie（通常浏览器会自动处理）

## 错误响应

### 401 未登录

```json
{
  "code": 401,
  "message": "未登录",
  "data": null
}
```

**说明**：该错误仅在需要登录的操作中返回（如创建投票、删除投票）

### 403 无权限

```json
{
  "code": 403,
  "message": "无权删除",
  "data": null
}
```

### 404 投票不存在

```json
{
  "code": 404,
  "message": "投票不存在",
  "data": null
}
```

### 400 已投过票

```json
{
  "code": 400,
  "message": "已经投过票了",
  "data": null
}
```

### 400 选项无效

```json
{
  "code": 400,
  "message": "选项无效",
  "data": null
}
```

## 投票流程

### 登录用户投票流程

1. 发送投票请求
2. 服务器验证用户身份和权限
3. 检查是否已投过票
4. 记录投票
5. 更新投票计数
6. 返回成功响应

### 路人投票流程

1. 发送投票请求
2. 服务器检查 `visitor_id` cookie
3. 如果没有，生成新的 `visitor_id`
4. 检查该 `visitor_id` 是否已投过票
5. 记录投票（使用 `visitor_id`）
6. 设置 `visitor_id` cookie（有效期30天）
7. 更新投票计数
8. 返回成功响应
